About Me

Up and coming Software Engineer

My name is Hailey Snell and I am a full stack software developer at Revature. I've always had a deep love for coding and problem solving which began as small challenges on various coding practice websites with friends and grew into getting a degree in Computer Science from the University of South Carolina. The first language I learned to use was Visual Basic, and I personally feel as though it was a very intuitive and straightforward language that anyone could use to get into programming. From there, I started to learn Python and Java on my own, and I would practice algorithm design using websites such as CodeWars.com which I still use to this day. In university, I was introduced to C++ as well as some other less common languages such as Prolog and Haskell.

Since then, working at Revature has taken my skills to the next level as I have since been introduced to and become proficient in C#, SQL, HTML, and Angular. University mainly focused on back-end programming, which made the experience here at Revature all the more valuable to create entire applications, and not just the logic behind them. Revature has also allowed for much more experience working as a team, and has taught me how to solve the problems that come with having multiple people working on the same project. Not only that, but it has also taught me to enjoy the process of coming together with other like minded individuals to create something great. Here you can find more info on my journey so far!

Banking Console App

My first project since starting training at Revature was a console banking app built in C#. This application allowed a user to login with credentials cross checked in SQL. From there users could do anything you could do at a real ATM including depositing and withdrawing. The application also supports admin users with a different list of options upon login.

My Goals

Security

Another career path I considered persuing was mechanical engineering. Something that was shared very early on when exploring mechanical engineering was that if I were to be assigned to build a bridge and a fault occurs because of my miscalculations, the blame is entirely on me and it's taken very seriously. That concept has since carried over to software development for me, in that if I write a program and users trust that program is secure, if any sort of data breach happens, the fault is mine. Because of this, I strive to create secure code and stay up to date on all the security news regarding the development tools that I use.

Software

As my first job, I worked for my local city doing GIS (Geographic Information Systems). In this job, I used a lot of software, but the main one was always ArcGIS. ArcGIS is a huge software developed by a company called ESRI. My boss at the time would allow me to take some "break days" to do some research on how the software was made. I learned that being an enterprise software, ArcGIS uses docker, kubernetes, and a lot of java. Since then, I've dreamed of working on a software of similar scale. Percet for me, it seems everything I'm learning at Revature is leading me in that direction.
